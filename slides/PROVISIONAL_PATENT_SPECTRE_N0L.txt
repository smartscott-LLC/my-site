FIELD OF THE INVENTION
This invention relates to asynchronous digital logic, self-timed inter-module communication, and
frequency-domain data processing systems implemented in reconfigurable hardware (FPGA)
and associated digital logic. More particularly, this invention relates to an architecture in which
frequency-encoded pulses — derived from the MIDI-mapped frequency vocabulary established
in the parent SPECTRE system — serve simultaneously as data carriers and as inter-module
handshake signals, collapsing the data path and the control path into a single physical signal
and eliminating the global clock tree from the inter-module communication layer.

BACKGROUND
1.1 The Global Clock Constraint
Every conventional synchronous digital system — including all current commercial FPGA
designs — operates under a global clock tree. The clock tree distributes a periodic timing signal
to every flip-flop in the design. All state transitions occur on clock edges. All combinational logic
must settle within one clock period (the critical path). The clock period is set by the slowest path
in the design.
This architecture imposes three fundamental costs:
Dead time: Logic that completes its computation in 0.3 ns of a 1.67 ns clock period (600 MHz)
sits idle for the remaining 1.37 ns, waiting for the next clock edge. This dead time can represent
30–80% of total potential throughput depending on the distribution of path delays across the
design.
Clock power: The clock tree toggles continuously — every register, every clock buffer, every
clock spine — at full frequency, even when no data is moving. Dynamic power consumption is
dominated by clock tree switching in large synchronous designs, often representing 30–50% of
total chip power.
Clock skew as scaling limit: As the number of connected processing nodes grows,
maintaining sub-nanosecond clock skew across all nodes becomes exponentially more difficult.
At board-to-board and system-to-system scales, a shared global clock is physically impossible
without specialized clock distribution hardware (e.g., White Rabbit PTP). Even with such
hardware, the shared clock remains a single point of failure and a fundamental scaling ceiling.

1.2 Asynchronous and Self-Timed Logic
Asynchronous digital logic — logic that operates without a global clock — has been studied
since the 1950s. Key paradigms include:

Muller handshake circuits: REQ/ACK two-phase or four-phase handshake protocols
between adjacent logic stages.
Null Convention Logic (NCL): Data is encoded in three states — NULL (no data),
DATA0 (logic zero), DATA1 (logic one). Threshold gates (THmn) fire only when a
complete set of input states has arrived. The NULL state between data words separates
successive computations and prevents wave collisions.
Wave pipelining: Multiple data waves traverse the same combinational logic
simultaneously. Registers between pipeline stages are eliminated. Throughput
improvement of up to 5.7× over synchronous pipelines has been demonstrated in
equalized-path implementations.
Delay-insensitive (DI) circuits: Circuits whose correct operation is guaranteed
regardless of the propagation delays of individual gates and wires, as long as delays are
finite and positive. DI circuits automatically adapt to silicon variation, temperature, and
supply voltage.

Despite decades of research, asynchronous logic has seen limited commercial FPGA adoption
because: (a) standard EDA tools (Vivado, Quartus) are designed for synchronous analysis; (b)
the REQ/ACK handshake signals require additional wires separate from the data path; and (c)
asynchronous designs are difficult to verify with standard simulation flows.

1.3 The SPECTRE Frequency Vocabulary
The parent invention SPECTRE (SES-003) established a 128-entry bijective frequency
vocabulary: the MIDI equal-temperament pitch standard maps each 7-bit data symbol to a
unique acoustic frequency between 8.18 Hz and 12,543.85 Hz. This vocabulary has been
validated as a lossless data encoding alphabet across nine FPGA device families.
A critical empirical observation from SPECTRE validation: cross-board timing variance is less
than 3 nanoseconds across nine device families spanning 28nm through 7nm-equivalent
process nodes. This is a remarkable result for a system in which the transmitting and receiving
boards run entirely independent clocks with no shared synchronization signal. The mechanism
is that SPECTRE's Goertzel-based decoder is inherently delay-insensitive: it detects which
frequency is present in a 480-sample window, not when (relative to any absolute time reference)
that frequency arrives. The frequency measurement is invariant to clock drift, temperature
variation, and silicon speed grade.
This observation establishes that SPECTRE's frequency vocabulary already implements
delay-insensitive inter-board communication as an emergent property of its encoding paradigm

— without having been designed to do so explicitly. SPECTRE N0L makes this emergent
property explicit, intentional, and architecturally foundational.

1.4 The Uncaptured Opportunity
No prior art is known that:
1.​ Collapses the data encoding signal and the inter-module handshake signal into a single
physical frequency-encoded pulse.
2.​ Applies the MIDI-mapped acoustic frequency vocabulary as a system-wide inter-module
communication protocol (not merely a data encoding alphabet).
3.​ Achieves delay-insensitive inter-board synchronization as a natural consequence of

4.​ Enables unlimited board-count scaling by replacing clock-based synchronization with
frequency-arrival-based wake-up.
5.​ Achieves zero-idle-power operation in reconfigurable hardware by making module
switching activity exactly proportional to incoming frequency wave rate.

SUMMARY OF THE INVENTION
SPECTRE N0L (No-clock / No Limits) is an asynchronous, self-timed, frequency-activated
inter-module communication architecture that extends the SPECTRE frequency vocabulary from
a data encoding layer to a universal system protocol layer. In SPECTRE N0L, every module in
the processing pipeline — and every board in a multi-board system — communicates
exclusively through SPECTRE frequency pulses. There is no global clock. There is no shared
synchronization signal. There is only: frequency arrives → module activates → processes →
emits next frequency → returns to dark state.
The central innovation is the collapse of data and control into a single physical signal. In
conventional asynchronous logic (e.g., NCL, Muller handshake circuits), data wires and
REQ/ACK handshake wires are separate. A data word is accompanied by a separate REQ
signal asserting "data is present," and a separate ACK signal returning "data has been
consumed." In SPECTRE N0L, the REQ signal is the data: the arrival of a frequency pulse at a
module's input simultaneously carries the 7-bit data payload AND asserts the request for the
receiving module to begin processing. The receiving module's completion of Goertzel frequency
detection simultaneously recovers the data payload AND generates the acknowledgment that
the transmission is complete.
The NULL state in SPECTRE N0L corresponds directly to the NULL state in Null Convention
Logic: the absence of any SPECTRE frequency pulse. A module in the NULL state is
consuming only static leakage power — no dynamic switching, no clock toggling, no idle

current. A module transitions from NULL to active in the same instant the first sample of an
incoming frequency pulse enters its Goertzel accumulator. This is the physical implementation of
zero-idle-power operation.
The wave pipeline emerges naturally: because each module activates independently on
frequency arrival and completes independently on Goertzel detection, multiple frequency waves
can be in transit through the pipeline simultaneously — one being detected by Module N while
Module N-1 is already emitting the next frequency and Module N+1 is already staging its
Goertzel accumulator. The pipeline stages are separated not by registers clocked on a shared
edge, but by the physical propagation delay of the frequency pulse through the transmission
medium.
Infinite board scaling follows directly: adding a board to the system requires only connecting it
to the frequency transmission medium. No clock distribution infrastructure is required. No PLL
lock sequence. No clock skew budget reallocation. The new board wakes up when a frequency
pulse arrives, processes it, and emits the next frequency. The system scales by the laws of
physics, not by the constraints of clock tree engineering.

DETAILED DESCRIPTION OF THE PREFERRED
EMBODIMENT
4.1 The Foundational Principle: One Signal, Three Roles
In SPECTRE N0L, a single frequency-encoded pulse simultaneously performs three roles that
are implemented as three separate signals in all prior art:
Role 1 — Data carrier: The frequency of the pulse identifies the 7-bit data payload via the
MIDI-mapped bijective LUT established in SPECTRE (SES-003). A pulse at 440 Hz (MIDI note

69) carries the 7-bit symbol 0x45. A pulse at 261.63 Hz (MIDI note 60, middle C) carries 0x3C.
The complete 128-entry mapping is inherited from SPECTRE with no modification.
Role 2 — Request (REQ) signal: The presence of any SPECTRE frequency pulse at a
module's input — detectable within the first few samples of the Goertzel accumulator as rising
spectral power — implicitly asserts REQ to the receiving module. The module transitions from
ST_NULL to ST_ACCUMULATE without any separate wire being asserted and without waiting
for a clock edge.
Role 3 — Implicit acknowledgment generation: The receiving module's completion of
Goertzel detection (ST_DETECT) is the implicit ACK. The transmitting module considers its
transmission acknowledged when the next frequency pulse arrives from downstream — either
directly as a separate feedback frequency or inferred from the pipeline's equalized-path timing.
In a unidirectional pipeline (the preferred embodiment), the ACK is implicit in the pipeline flow
and no explicit acknowledgment wire is required.
This collapse from three signals to one is the primary engineering innovation of SPECTRE N0L.
It reduces inter-module interface complexity to a single differential pair (LVDS or equivalent) per
directed connection.

The 7-bit data payload is recovered by Goertzel detection of the FREQ_P/FREQ_N signal. The

4.3 Novel Contribution 2 — Delay-Insensitive Cross-Platform
Synchronization as Designed Property
The SPECTRE parent system (SES-003) demonstrated empirically that cross-board timing
variance is less than 3 nanoseconds across nine FPGA device families with no shared clock.
SPECTRE N0L makes this property architectural and intentional rather than emergent.
The mechanism: Goertzel frequency detection is mathematically a measurement of spectral
power at a target frequency over N samples. The measurement result — the detected MIDI note
— is independent of: the absolute time at which the frequency pulse arrived, the phase offset of
the receiving module's sampling clock relative to the transmitting module's clock, the
temperature and supply voltage of the receiving silicon (which affect gate delays and therefore
sampling clock accuracy), and the speed grade of the device family.
The only requirement for correct detection is that N = 480 samples are accumulated at 96,000
samples per second. The sample clock frequency must be stable over the 5 ms window
duration — a requirement met by any FPGA PLL with standard jitter specifications.
This property is delay-insensitive by the formal DI definition: the system's logical correctness is
guaranteed for any finite positive propagation delay. Adding a board to the pipeline introduces
additional propagation delay but does not change the correctness of any module's Goertzel
detection. The system adapts automatically.

4.4 Novel Contribution 3 — Zero-Idle-Power Operation
In a synchronous clocked system, dynamic power consumption is:
P_dynamic = α × C × V² × f_clock
where α is the switching activity factor (fraction of flip-flops and logic gates that switch per clock
cycle). Even when no data is being processed, α > 0 because the clock tree itself toggles every

flip-flop's clock input at every clock edge. In large designs, the clock tree alone can represent α
≈ 0.3–0.5 of total switching activity.
In a SPECTRE N0L module, there is no clock tree. When no frequency pulse is present
(ST_NULL state), the only power consumed is static leakage through transistor gate oxides.
Dynamic switching activity is zero. Dynamic power is zero. The module is "dark."
When a frequency pulse arrives, the Goertzel accumulator begins switching. Dynamic power
scales linearly with the incoming pulse frequency:
P_dynamic_N0L = α_goertzel × C × V² × f_pulse_rate
where f_pulse_rate is the rate of incoming frequency pulses (at most 96,000 per second for a
480-sample window at 96 kHz) and α_goertzel is the switching activity of the Goertzel
accumulator only.
The zero-idle-power property enables a new class of system architecture: a pipeline of N boards
can remain fully dark until a single triggering frequency pulse is injected at Board 1. The pulse
propagates through the pipeline, activating each board only during its active processing window,
then returning each board to the dark state. Total energy consumed per data unit is bounded
and proportional to data volume — not to wall-clock time.

4.5 Novel Contribution 4 — Wave Pipeline Without Registers
In a standard synchronous pipeline, N pipeline stages require N-1 register stages (flip-flop
banks) to hold data at stage boundaries while downstream stages complete their work. Each
register stage consumes FPGA flip-flop resources and introduces one clock cycle of latency.
In SPECTRE N0L, register stages between pipeline modules are eliminated. Multiple frequency
waves can be in transit through the pipeline simultaneously, separated by the physical
propagation delay of the transmission medium (approximately 5 ns per meter for electrical
signals in PCB trace or coaxial cable; approximately 3.3 ns per meter in fiber optic).
The equalized-path condition: For wave pipelining to be correct, the minimum and maximum
propagation delays through each pipeline stage must be equalized — that is, the difference
between the fastest and slowest path through any stage must be less than one wave spacing. In
SPECTRE N0L, each module's processing time is deterministic and equal: 480 Goertzel
accumulation cycles + detection + emission = a fixed number of clock cycles at the local
sampling clock. This determinism is an inherent property of the Goertzel algorithm applied to a
fixed-length window. The equalized-path condition is satisfied by design.
Research in conventional wave pipelining demonstrates throughput improvements of up to 5.7×
over synchronous pipelines with equivalent logic depth. SPECTRE N0L achieves this

improvement without any of the specialized EDA tooling required for conventional wave pipeline
design, because the pipeline stages are separated by physical transmission medium (wire/fiber)
rather than by on-chip combinational logic paths.

4.6 Novel Contribution 5 — Universal Frequency Protocol: Every Module
Speaks SPECTRE
In the parent SPECTRE system (SES-003), the frequency vocabulary is used at one layer: the
data encoding interface between the encoder and the WAV storage / Goertzel decoder. Modules
internal to the FPGA communicate via standard synchronous RTL signals (std_logic_vector,
valid/ready handshake, AXI4-Lite, etc.).
In SPECTRE N0L, the MIDI-mapped 128-frequency vocabulary is the universal protocol for ALL
inter-module communication. Every module in the system — regardless of its function (oscillator,
hub, extent_tree, stack_reader, CRC engine, PCIe interface) — has a SPECTRE N0L interface:
it receives frequency pulses as input and emits frequency pulses as output. The module's
function determines which frequency it emits in response to which incoming frequency, but the
physical interface is identical for every module.
The architectural consequence: Every module and every board in the system is natively
interoperable. A board does not need to know whether it is communicating with an oscillator
module or a CRC engine module. It receives a frequency, processes it, emits the next
frequency. The system can be reconfigured by changing the mapping between incoming and
outgoing frequencies at each module — a software operation on the frequency-routing table —
without any hardware change.
The "fleet" scaling property: A fleet of N boards connected by LVDS differential pairs all
running SPECTRE N0L are natively interoperable from the moment they are powered. Board N
does not need to be configured to communicate with Board M — both already speak SPECTRE.
Adding a board is equivalent to adding a native speaker to a conversation: no translation layer
required.

4.7 Novel Contribution 6 — The NULL State as Architecture
In Null Convention Logic (NCL), the NULL state is a data encoding level that separates
successive data words in a bundled data channel. It is a protocol artifact — a separator that
must be explicitly generated and detected.
In SPECTRE N0L, the NULL state is the physical absence of any frequency pulse on the
transmission medium. It requires no explicit generation (a module simply stops transmitting), no
explicit detection (a module simply observes that no Goertzel bin has sufficient power), and no
additional protocol overhead. The NULL state is free: it costs nothing to generate, nothing to
detect, and nothing to transmit.

This property — NULL as the absence of signal rather than as an explicit signal level — is
unique to frequency-domain encoding. In binary encoding, silence (no signal) is ambiguous: it
could mean NULL, or it could mean a transmission error, or it could mean the line is idle. In
SPECTRE N0L, silence has unambiguous meaning: no data is present. A module in a
silent-input state remains in ST_NULL. There is no ambiguity.
The NULL state in SPECTRE N0L therefore simultaneously implements: wave separation
(prevents wave collisions in the wave pipeline), idle detection (distinguishes no-data from data),
and power gating (the module's switching activity drops to zero in the NULL state automatically,
without any power management logic).

4.8 Novel Contribution 7 — Raw LVDS Pin Interface: Bypassing
Transceiver Clock Recovery
Standard FPGA high-speed transceivers (e.g., AMD GTY, GTM) incorporate Phase-Locked
Loops (PLLs) and Clock Data Recovery (CDR) circuits that re-synchronize received serial data
into the local clock domain. This is the correct behavior for protocols that require synchronous
processing (PCIe, Ethernet, JESD204B). For SPECTRE N0L, it is exactly the wrong behavior:
CDR would force the received frequency pulse back into a clocked domain, destroying the
asynchronous self-timed property.
SPECTRE N0L bypasses high-speed transceivers and uses raw LVDS differential I/O pins with
IBUFDS (input differential buffer) at the receiver and OBUFDS (output differential buffer) at the
transmitter. The received differential signal feeds directly into the Goertzel accumulator's sample
input. No CDR, no PLL lock sequence, no 8b/10b framing overhead.
LVDS on UltraScale+ HR I/O banks supports signaling rates up to approximately 1.6 Gbps in
raw mode. For SPECTRE N0L's 96 kHz sample rate (well below this limit), LVDS provides
ample bandwidth with sub-nanosecond signal integrity on PCB traces and with standard
SFP/optical transceivers for inter-board connections.
The Xilinx IDELAY/ODELAY primitives (IDELAYE3/ODELAYE3 on UltraScale+) provide per-pin
delay adjustment with 2 ps resolution over a 1.25 ns range. These primitives enable per-board
wave pipeline deskewing: if Board M's Goertzel window completes slightly early or late relative
to Board M+1's accumulation start, ODELAY at Board M's output and IDELAY at Board M+1's
input can be tuned to compensate. This is the hardware equivalent of equalizing the wave
pipeline paths across a multi-board system.

SPECTRE N0L is the culminating expression of the Shimmers · Eidolon · SPECTRE system.
When all three components are realized in N0L:

The emergent system property of full N0L integration: In the fully realized SES N0L system,
there is no global clock, no shared synchronization signal, and no inter-module wiring beyond

ABSTRACT
SPECTRE N0L (No-clock / No Limits) is an asynchronous self-timed inter-module
communication architecture extending the SPECTRE frequency-domain encoding paradigm
(SES-003) from the data layer to the system timing and control layer. In SPECTRE N0L,
frequency-encoded pulses from the MIDI-mapped 128-frequency vocabulary serve
simultaneously as data carriers, inter-module request signals, and activation triggers,
eliminating the need for separate handshake wires and eliminating the global clock tree from the
inter-module communication layer. The receiving module's spectral frequency detection
completion constitutes an implicit acknowledgment. Modules consume zero dynamic power
when no frequency pulse is present. Multiple data waves propagate through the pipeline
simultaneously without register stages. Cross-board timing variance below 3 nanoseconds is
